# 용어사전 업그레이드 요청

아래 키워드를 `용어 사전` 또는 관련 본문 영역에 추가·보완해 주세요. 대상 독자는 블록체인·금융·개발 비전공자입니다. 전문 용어를 그대로 나열하지 말고, 독자가 실제 상황을 떠올릴 수 있도록 쉬운 설명과 예시를 함께 넣어 주세요.

## 작성 원칙

- 넘버링 순서에 맞추지 말고, 의미가 가까운 키워드끼리 묶어 주세요.
- 각 용어는 `한 줄 뜻` + `쉬운 설명` + `예시` + 필요한 경우 `주의점` 순서로 작성해 주세요.
- 설명은 은행, 자판기, 금고, 포인트 앱, 상품권처럼 일상 비유를 활용해 주세요.
- 투자·수익형 용어에는 손실 가능성, 출금 조건, 청산, 스마트 계약 위험, 오프체인 법적 위험을 함께 적어 주세요.
- `스마트 계약`, `스마트 컨트랙트`, `smart contract`는 같은 개념으로 설명하되, `스마트 컨트렉트`는 비표준 표기라는 점을 자연스럽게 정리해 주세요.
- `갤럭시`가 아니라 [GALXE](https://www.galxe.com/)입니다. 퀘스트·크리덴셜·에어드랍 플랫폼으로 수정해 주세요.

## A부 핵심용어로 올릴 기준

용어사전에만 두지 말고, 아래에 해당하는 용어는 A부 `꼭 알아야 할 핵심 용어`에도 올려 주세요.

- 뒤 챕터의 월렛 연결, 서명, 서비스 비교를 이해하는 데 필요한 기본 문법
- 사용자가 실제로 송금·연결·서명·조회할 때 바로 마주치는 단어
- 특정 상품명보다 범용 개념에 가까운 단어

### A부로 승격할 용어

- `핫 월렛 / 콜드 월렛`
- `임베디드 월렛`
- `스마트 월렛`
- `Web3`
- `dApp`
- `EVM`
- `DeFi`
- `거래 해시 / 블록 탐색기`

### 용어사전에 남기는 것이 더 좋은 용어

- `DeFi Gateway`, `자산 어그리게이터`, `Vault`, `SuperEarn`, `Morpho`, `Pendle`, `Yield8`, `Yearn Finance`, `Beefy Finance`
- `스테이킹`, `레버리지 스테이킹`, `DEX LP풀`, `Curve LP`, `담보대출 프로토콜`, `RWA 펀드`
- `Kaia 체인`, `GALXE`, `신규토큰 에어드랍`, `Zora Rewards`, `Sweat Economy`, `StickK(Web2)`, `온체인 로또`, `P2P 제로섬`

이 용어들은 중요하지만 상품·전략·서비스 성격이 강하므로, A부에서는 `DeFi`와 `dApp`의 예시로 짧게 언급하고 상세 설명은 용어사전에 둡니다.

```html
<tr class="gl-row"><td>스마트 계약 / 스마트 컨트랙트</td><td>조건이 맞으면 자동 실행되는 블록체인 프로그램</td></tr>
<tr class="gl-detail"><td colspan="2"><div class="gl-inner"><strong>스마트 계약</strong>은 종이 계약서가 아니라 블록체인 위에 올려 둔 공개 프로그램입니다. 돈을 넣으면 음료가 나오는 자판기처럼, 조건이 맞으면 정해진 행동을 자동으로 실행합니다.<br><strong>예시:</strong> NFT 구매 계약은 "구매자가 돈을 보내면 판매자에게 돈을 보내고 NFT 소유권을 구매자에게 옮긴다"는 규칙을 실행합니다. 악성 계약에 서명하면 자산 이동 권한을 넘길 수 있으므로 서명 내용을 확인해야 합니다.</div></td></tr>
```

## 최종 확인

- 용어사전의 모든 상세 설명에 최소 1개 이상의 `예시:` 문장이 있어야 합니다.
- `GALXE` 표기가 본문, 서비스 비교, 용어사전, 출처에서 일관되게 사용되어야 합니다.
- 펼침형 용어사전 구조는 `gl-row`와 `gl-detail`이 번갈아 나오도록 유지해 주세요.
