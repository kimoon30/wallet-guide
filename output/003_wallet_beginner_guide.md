# 디지털 월렛 초보자 가이드

> **한 줄 소개**
> 월렛은 코인을 담는 앱이 아닙니다. 블록체인 위 내 자산을 움직이도록 허락하는 **키와 서명 권한을 관리하는 도구**입니다.

`대상: 완전 초보자·비전공자` · `목표: 실수로 자산을 잃지 않는 기본 판단력 만들기` · `읽는 시간: 30~45분`

---

## 0. 10분 요약 — 이것만 알아도 사고 확률이 줄어듭니다

처음에는 모든 용어를 외우려고 하지 않아도 됩니다. 아래 여섯 문장을 먼저 기억하면 됩니다.

1. **월렛은 돈을 보관하는 창고가 아니라 키를 관리하는 도구입니다.**
2. **지갑 주소는 알려줘도 되지만, 개인키와 시드 문구는 절대 알려주면 안 됩니다.**
3. **코인과 토큰은 다릅니다. 토큰을 보내려면 수수료용 코인이 따로 필요할 수 있습니다.**
4. **같은 USDT라도 이더리움, 트론, 솔라나처럼 네트워크가 다르면 다른 자산처럼 다뤄야 합니다.**
5. **이해하지 못한 연결, 서명, 승인은 거절합니다.**
6. **처음에는 반드시 잃어도 되는 소액으로 테스트합니다.**

> **첫 회독 목표**
> 월렛을 능숙하게 쓰는 것이 아니라, 위험한 순간에 멈출 수 있는 기준을 갖는 것입니다.

---

## 1. 월렛은 무엇인가

### 한 문장 정의

디지털 월렛은 블록체인에 기록된 내 디지털 자산을 **확인, 전송, 사용**할 수 있도록 키와 서명을 관리하는 도구입니다.

코인이나 NFT가 월렛 앱 안에 파일처럼 들어 있는 것은 아닙니다. 자산과 거래 내역은 블록체인에 기록되고, 월렛은 그 자산을 움직일 수 있는 **권한**을 관리합니다.

### 쉬운 비유

| 비유 | 실제 개념 |
|---|---|
| 공동 장부 | 블록체인 |
| 집 주소 | 지갑 주소 |
| 문을 여는 비밀 열쇠 | 개인키 |
| 열쇠 꾸러미 | 월렛 |
| 장부상 보유 자산 | 코인, 토큰, NFT |

주소를 아는 것만으로는 자산을 옮길 수 없습니다. 하지만 개인키나 시드 문구를 가진 사람은 거래에 서명할 수 있으므로 사실상 그 지갑을 통제합니다.

### 은행 앱과 다른 점

| 구분 | 은행·금융 앱 | 일반적인 블록체인 월렛 |
|---|---|---|
| 자산 기록 장소 | 금융회사 서버 | 블록체인 네트워크 |
| 거래 승인 | 금융회사 시스템 | 개인키 보유자의 서명 |
| 비밀번호 분실 | 본인 확인 후 재설정 가능 | 시드 문구 없으면 복구 어려움 |
| 잘못 보낸 돈 | 기관을 통한 조치 가능성 | 되돌리기 어려운 경우 많음 |

> **핵심 정리**
> 자산은 블록체인에 있고, 권한은 월렛이 관리합니다.

## 2. 꼭 알아야 할 핵심 용어

### 지갑 주소

자산을 받을 때 알려주는 공개 식별값입니다. 계좌번호와 비슷하게 생각해도 됩니다.

- 이더리움 주소는 보통 `0x`로 시작합니다.
- 솔라나, 비트코인 등은 주소 형식이 다릅니다.
- 주소는 공개해도 되지만, 네트워크와 주소 형식이 맞는지 확인해야 합니다.

### 개인키와 시드 문구

개인키는 거래에 서명하는 권한입니다. 시드 문구는 여러 개인키를 복구할 수 있는 마스터키입니다.

| 항목 | 공유 가능 여부 | 설명 |
|---|---|---|
| 지갑 주소 | 가능 | 자산을 받을 때 쓰는 공개값 |
| 공개키 | 일반적으로 공개 가능 | 주소를 만드는 데 쓰이는 공개 정보 |
| 개인키 | 절대 불가 | 거래 서명 권한 |
| 시드 문구 | 절대 불가 | 지갑 전체 복구용 마스터키 |

시드 문구는 사진, 클라우드, 메신저, 이메일, 메모 앱에 저장하지 않는 것이 안전합니다. 종이에 적어 오프라인으로 보관하는 편이 낫습니다.

### 코인, 토큰, NFT

| 구분 | 뜻 | 예시 |
|---|---|---|
| 코인 | 네트워크의 기본 화폐이자 수수료 결제 수단 | ETH, SOL, BTC |
| 토큰 | 특정 네트워크 위에서 발행된 자산 | USDT, USDC |
| NFT | 각각 고유한 토큰 | 디지털 아트, 게임 아이템 |

초보자가 자주 겪는 문제는 **토큰은 있는데 수수료용 코인이 없어서 전송하지 못하는 상황**입니다. 예를 들어 이더리움 네트워크의 USDT를 보내려면 수수료로 ETH가 필요합니다.

### 네트워크

블록체인은 하나가 아닙니다. 이더리움, 솔라나, 비트코인, 폴리곤처럼 각 네트워크는 규칙과 수수료가 다릅니다.

같은 USDT라도 네트워크가 다르면 전송 경로가 달라집니다. 거래소에서 출금하거나 월렛으로 받을 때는 항상 **자산 이름과 네트워크를 함께 확인**해야 합니다.

## 3. 거래는 어떻게 일어나는가

대부분의 거래는 아래 흐름을 따릅니다.

```text
주소 확인 -> 수량 확인 -> 수수료 확인 -> 서명 -> 네트워크 기록
```

### 예시: ETH를 다른 주소로 보내기

1. 받는 주소를 입력합니다.
2. 보낼 수량을 입력합니다.
3. 가스비, 즉 수수료를 확인합니다.
4. 전송 내용을 확인하고 서명합니다.
5. 네트워크에 기록되면 상대 주소 잔액이 증가합니다.

가장 위험한 순간은 보통 **서명 직전**입니다. 서명은 “이 거래나 요청에 동의한다”는 의미이기 때문입니다.

### 예시: NFT 마켓에서 NFT 구매하기

1. 월렛을 사이트에 연결합니다.
2. NFT를 선택합니다.
3. 가격과 가스비가 표시됩니다.
4. 구매 내용을 확인하고 서명합니다.
5. 스마트 계약이 실행되고 소유권이 내 주소로 이전됩니다.

월렛 연결만으로 개인키가 전달되지는 않습니다. 하지만 연결 후 뜨는 서명 요청이 위험할 수 있습니다.

## 4. 연결, 서명, 승인, 전송의 차이

초보자는 이 네 가지를 반드시 구분해야 합니다.

| 행동 | 무슨 뜻인가 | 위험도 |
|---|---|---|
| 지갑 연결 | 사이트가 내 주소를 볼 수 있음 | 낮음~중간 |
| 서명 | 특정 메시지나 거래 요청에 동의함 | 중간~높음 |
| 토큰 승인 | 특정 계약이 내 토큰을 사용할 권한을 얻음 | 높음 |
| 전송 | 실제 자산이 다른 주소로 이동함 | 높음 |

연결 해제와 승인 취소도 다릅니다.

| 구분 | 의미 |
|---|---|
| 연결 해제 | 사이트가 내 지갑 주소를 다시 쉽게 보지 못하게 함 |
| 승인 취소 | 스마트 계약에 준 토큰 사용 권한을 회수함 |

안 쓰는 dApp 연결과 토큰 승인은 주기적으로 정리하는 습관이 좋습니다.

## 5. 서명 팝업에서 멈춰야 하는 문구

아래 표현이 보이면 초보자는 일단 멈추는 것이 안전합니다.

| 문구 | 왜 조심해야 하나 |
|---|---|
| `approve` | 토큰 사용 권한을 주는 요청일 수 있음 |
| `setApprovalForAll` | NFT 전체 권한을 넘기는 요청일 수 있음 |
| `unlimited approval` | 무제한 사용 권한을 주는 요청일 수 있음 |
| `blind signing` | 내용을 명확히 보지 못한 채 서명하는 상황일 수 있음 |
| “이 메시지에 서명하세요” | 무엇에 동의하는지 불분명할 수 있음 |
| “권한을 부여합니다” | 자산 사용 권한을 넘기는 요청일 수 있음 |

### 팝업 문구 예시

아래 예시는 실제 서비스마다 표현이 조금씩 다를 수 있습니다. 핵심은 **내가 무엇에 동의하는지 설명할 수 있는가**입니다.

| 예시 문구 | 초보자 판단 |
|---|---|
| `Approve unlimited spending of USDT` | 거절. 무제한 승인일 수 있습니다. |
| `Set approval for all NFTs` | 거절. NFT 전체 권한을 넘길 수 있습니다. |
| `Enable blind signing` | 거절. 내용을 보지 못한 채 서명할 수 있습니다. |
| `Sign this message to continue` | 내용이 불분명하면 거절합니다. |
| `Send 0.01 ETH to 0x1234...abcd` | 주소, 수량, 수수료가 내가 요청한 내용과 맞는지 확인합니다. |
| `Network fee: 0.0003 ETH` | 수수료 표시입니다. 전송 자산과 수수료 코인이 다를 수 있습니다. |

### 초보자 판단 질문

서명 팝업이 뜨면 아래 질문에 답해봅니다.

| 질문 | 답이 애매하면 |
|---|---|
| 내가 방금 직접 요청한 행동인가? | 거절 |
| 어떤 자산이 얼마만큼 이동하는지 이해했나? | 거절 |
| 사이트 주소가 공식 주소인지 확인했나? | 거절 |
| 이 지갑에 잃으면 곤란한 금액이 들어 있나? | 연결하지 않음 |
| 권한 요청 범위가 이해되는가? | 거절 |

> **원칙**
> 설명할 수 없는 버튼, 이해하지 못한 서명, 출처가 불분명한 링크는 누르지 않습니다.

## 6. 대표 서비스는 유형으로 구분합니다

월렛 관련 서비스는 모두 같은 경쟁 서비스가 아닙니다. 먼저 유형을 봐야 합니다.

| 서비스 | 한 줄 설명 | 유형 |
|---|---|---|
| MetaMask | 여러 Web3 서비스에 연결하는 범용 월렛 | 소프트웨어 월렛 |
| Phantom | 솔라나 중심에서 멀티체인으로 확장한 월렛 | 소프트웨어 월렛 |
| D'CENT | 개인키를 별도 기기에 보관하는 월렛 | 하드웨어 월렛 |
| GALXE | 미션, 크리덴셜, 보상 조건을 운영하는 플랫폼 | Web3 퀘스트 플랫폼 |
| 마이비(MyB) | 리워드 앱 안에 지갑 기능이 결합된 서비스 | 임베디드 월렛 |
| Polymarket | 월렛을 연결해 쓰는 예측시장 서비스 | dApp |

### 고를 때 보는 기준

1. 키를 누가 관리하는가
2. 어떤 네트워크를 지원하는가
3. 복구 방식은 무엇인가
4. 수수료는 누가 부담하는가
5. 지역 제한이나 규제 이슈가 있는가
6. 큰 금액을 보관할 지갑과 실험용 지갑을 분리했는가

처음에는 하나의 지갑에 모든 자산을 넣기보다, **학습용 지갑과 보관용 지갑을 분리**하는 것이 좋습니다.

## 7. 처음 실습하는 순서

처음부터 큰 금액을 옮기지 않습니다. 아래 순서로 익숙해지는 것이 안전합니다.

### 1일차: 개념만 이해하기

- 월렛이 키 관리 도구라는 점을 설명해봅니다.
- 주소, 개인키, 시드 문구의 차이를 정리합니다.
- 코인, 토큰, NFT, 가스비의 차이를 확인합니다.

완료 기준: “월렛은 코인을 담는 앱이 아니라 서명 권한을 관리하는 도구”라고 말할 수 있습니다.

### 2일차: 거래 흐름 관찰하기

- 실제 자산을 옮기지 않고 전송 화면 예시를 찾아봅니다.
- 주소, 수량, 수수료, 네트워크가 어디에 표시되는지 확인합니다.

완료 기준: 전송 전 확인해야 하는 항목을 말할 수 있습니다.

### 3일차: 위험 행동 구분하기

- 시드 문구 노출, 지갑 연결, 토큰 승인, 오송금의 차이를 정리합니다.
- 이해하지 못한 서명은 거절한다는 기준을 세웁니다.

완료 기준: “연결 해제”와 “승인 취소”가 다르다는 점을 설명할 수 있습니다.

### 4일차: 서비스 유형 구분하기

- MetaMask, Phantom, D'CENT, GALXE, 마이비, Polymarket이 서로 어떤 유형인지 구분합니다.

완료 기준: 월렛, 플랫폼, 임베디드 월렛, dApp을 구분할 수 있습니다.

### 5일차: 소액 테스트 준비하기

- 공식 사이트나 공식 앱스토어 링크를 확인합니다.
- 학습용 지갑을 따로 만들 계획을 세웁니다.
- 실제 사용 전 소액 테스트 전송 계획을 세웁니다.

완료 기준: 큰 금액이 들어 있지 않은 별도 지갑으로만 실습할 준비가 됩니다.

## 8. 사고가 의심될 때 30초 분류

문제가 생겼을 때는 먼저 어떤 사고인지 분류합니다.

| 먼저 물어볼 질문 | 판단 |
|---|---|
| 시드 문구나 개인키를 입력, 전송, 촬영했나? | 기존 지갑은 안전하지 않음. 새 지갑을 만들고 자산 이동 필요 |
| 모르는 사이트에 연결만 했나? | 연결 해제 후 토큰 승인 여부를 별도 확인 |
| 서명이나 승인을 눌렀나? | 거래 해시와 승인 내역 확인. 필요하면 승인 취소 |
| 잘못된 주소나 네트워크로 보냈나? | 거래소나 수신 서비스 지원팀에 문의. 개인 지갑 전송은 복구 어려움 |

시드 문구가 노출된 지갑은 다시 안전해지지 않습니다. 새 지갑을 만들고, 남은 자산을 옮기는 편이 안전합니다.

---

## 9. 바로 실행할 수 있는 체크리스트

### 읽은 후 자기 점검

- [ ] 월렛이 코인을 보관하는 앱이 아니라 키 관리 도구임을 설명할 수 있다
- [ ] 지갑 주소와 시드 문구의 차이를 설명할 수 있다
- [ ] 지갑 연결과 토큰 승인이 다르다는 점을 설명할 수 있다
- [ ] 이해하지 못한 서명 요청은 거절해야 한다는 기준을 세웠다

### 처음 월렛을 만들 때

- [ ] 공식 사이트나 공식 앱스토어에서만 설치한다
- [ ] 시드 문구를 종이에 오프라인으로 기록한다
- [ ] 사진, 클라우드, 메신저, 이메일에 시드 문구를 저장하지 않는다
- [ ] 키 관리 주체, 지원 네트워크, 복구 방식을 확인한다

### 자산을 보낼 때

- [ ] 받는 네트워크가 맞는지 확인한다
- [ ] 주소를 복사해 붙여넣고 앞뒤 글자를 다시 확인한다
- [ ] 수수료용 코인이 지갑에 있는지 확인한다
- [ ] 본 전송 전에 소액 테스트 전송을 먼저 한다

### Web3 서비스에 연결하거나 서명할 때

- [ ] 사이트 주소와 평판을 확인한다
- [ ] 연결 목적을 이해했다
- [ ] 서명 팝업의 거래 내용과 권한 요청을 읽었다
- [ ] 불분명하면 거절한다
- [ ] 안 쓰는 dApp 연결과 토큰 승인을 주기적으로 정리한다

## 10. 초보자 최종 판단 기준

아래 네 문장에 답할 수 있으면 첫 회독 목표는 달성한 것입니다.

| 질문 | 답 |
|---|---|
| 월렛은 무엇인가? | 블록체인 위 자산을 움직이는 키와 서명을 관리하는 도구 |
| 절대 공유하면 안 되는 것은? | 개인키와 시드 문구 |
| 실제 사용에서 가장 먼저 멈춰야 하는 순간은? | 전송, 승인, 서명 직전 |
| 이해하지 못한 서명 팝업은 어떻게 해야 하나? | 거절 |

## 빠른 용어 사전

| 용어 | 한 줄 뜻 |
|---|---|
| 블록체인 | 여러 컴퓨터가 함께 보관하는 분산 거래 장부 |
| 월렛 | 자산을 움직이는 키와 서명을 관리하는 도구 |
| 지갑 주소 | 자산을 받을 때 공개하는 식별값 |
| 개인키 | 거래에 서명할 수 있는 권한 |
| 시드 문구 | 지갑 전체를 복구할 수 있는 마스터키 |
| 코인 | 네트워크의 기본 화폐이자 수수료 결제 수단 |
| 토큰 | 특정 네트워크 위에서 발행된 자산 |
| NFT | 각각 고유한 토큰 |
| 가스비 | 거래를 네트워크에 기록하기 위해 내는 수수료 |
| dApp | 월렛을 연결해 사용하는 블록체인 기반 서비스 |
| 스마트 계약 | 조건이 충족되면 자동 실행되는 블록체인 프로그램 |
| 토큰 승인 | 특정 계약이 내 토큰을 사용할 수 있도록 허락하는 것 |
| 승인 취소 | 이전에 준 토큰 사용 권한을 회수하는 것 |
| 콜드 월렛 | 개인키를 인터넷과 분리해 보관하는 월렛 |
| 임베디드 월렛 | 앱 내부에 내장되거나 연동된 월렛 |
