# 리워드 앱 시장과 마이비 분석 가이드

> **한 줄 소개**
> 리워드 앱은 “공짜로 돈을 주는 앱”이 아닙니다. 기업이 원하는 행동의 대가 일부를 사용자에게 돌려주고, 남는 수익과 데이터 품질로 지속 가능성을 증명해야 하는 사업입니다.

`대상: 서비스기획자·사업기획자` · `난이도: 중급~고급` · `범위: 리워드 앱 구조, 수익모델, 지표, 주요 사례, 마이비 분석`

---

## 0. 읽기 전 기준

리워드 앱을 볼 때는 다운로드 수나 가입자 수만으로 판단하면 안 됩니다. 아래 세 가지를 구분해야 합니다.

| 구분 | 의미 | 왜 구분해야 하나 |
|---|---|---|
| 공개 사실 | 보도자료, 공식 페이지, 앱스토어, 공시 등에서 확인 가능한 내용 | 검증의 출발점 |
| 운영사 발표치 | 회사가 직접 발표한 가입자, 성과, 목표 | 유용하지만 비교 기준과 산식 확인 필요 |
| 미공개 지표 | DAU, MAU, 리텐션, 앱 단독 매출, 보상비, 공헌이익 | 사업성 판단의 핵심이지만 외부에서 알기 어려움 |

> **분석 원칙**
> 공개되지 않은 서비스별 매출, 활성 사용자 수, 비용 구조는 임의로 계산하지 않습니다. 운영사 전체 실적과 개별 앱 실적도 반드시 구분합니다.

---

## 1. 리워드 앱이란 무엇인가

**핵심 메시지:** 리워드 앱은 무료로 돈을 주는 곳이 아니라, **기업이 원하는 행동의 대가 일부를 사용자에게 지급하는 사업**입니다.

사용자가 보상을 받는 행동은 광고 시청, 설문/퀴즈, 걷기, 출석, 앱 설치/가입, 쇼핑, 친구 초대, 게임 등입니다. 포인트는 현금, 상품권, 간편결제, 할인 등으로 사용됩니다.

예를 들어 쇼핑 앱이 신규 회원 1명에 5,000원을 지불하면, 리워드 플랫폼은 그중 일부를 사용자에게 주고 나머지로 운영비와 수수료를 충당합니다.

| 사용자 관점 | 기업 관점 |
|---|---|
| 남는 시간과 일상 행동을 소액 가치로 전환합니다. 보상까지 시간이 짧고, 포인트 가치가 명확하고, 현금화가 쉽고, 반복 이유가 있어야 합니다. | 돈을 나눠주려는 것이 아니라 광고 노출/클릭, 설치/가입, 설문 응답, 쇼핑·금융 유도, 재방문, 행동 데이터 수집을 위해 운영합니다. |

### 돈과 데이터의 흐름

```text
[돈]
광고주 캠페인 의뢰
-> 플랫폼이 미션 제공
-> 사용자 수행
-> 광고주가 결과에 따라 비용 지급
-> 일부를 사용자 보상·제휴 비용으로 지급
-> 남은 금액으로 인건비·서버비·마케팅비·수익 충당

[데이터]
사용자의 취향·구매·반응
-> 플랫폼 분석
-> 누구에게 어떤 광고를 보일지 결정
-> 광고 성과 개선
```

타기팅이 정확할수록 광고주는 같은 비용으로 더 많은 성과를 얻습니다. 다만 보상 때문에 사용자가 사실과 다르게 답하면 데이터 가치가 떨어집니다.

**챕터 요약:** 보상의 재원은 광고·조사·제휴 비용입니다. 그래서 데이터 품질과 반복 참여가 사업의 생명선입니다.

## 2. 유형, 수익모델, 핵심 성과지표

**핵심 메시지:** 다운로드 수가 아니라 **활성도, 재구매, 공헌이익**으로 사업성을 봅니다.

### 서비스 유형

| 유형 | 핵심 행동 | 대표 서비스 |
|---|---|---|
| 설문·취향 데이터형 | 질문에 답하고 적립 | 마이비, 오베이 |
| 행동·미션 리워드형 | 걷기, 출석, 위치, 잠금화면 등 일상 행동 | 캐시워크, 돈이돼지 |
| 자산·토큰 리워드형 | 지갑, 퀴즈, 미션으로 코인/금 보상 | 리워디, 코퀴즈 |
| 슈퍼앱 혜택형 | 금융·결제·쇼핑 안에 리워드 내장 | 토스 혜택, 토스 만보기 |

슈퍼앱은 리워드 자체에서 이익이 안 나도 금융, 결제, 쇼핑 등 다른 서비스에서 비용을 회수할 수 있습니다. 그래서 독립 리워드 앱보다 보상 경쟁에서 유리할 수 있습니다.

### 주요 수익모델

| 모델 | 설명 | 주의점 |
|---|---|---|
| 프로그래머틱 광고 | 실시간 자동 광고 구매·배치 | 물량은 안정적이나 단가 변동 |
| 직접 광고 | 브랜드와 직접 계약하는 캠페인 | 단가는 높지만 영업·제작 필요 |
| CPA | 설치, 가입, 구매 등 행동 완료 시 과금 | 부정 설치·허위 전환 방지 필요 |
| 설문·패널 사업 | 응답 데이터를 기업 리서치에 활용 | 응답 품질 관리가 핵심 |
| 제휴 커머스 | 쇼핑 전환에 따라 수익 배분 | 반품·취소 시 포인트 회수 필요 |
| 외부 광고 네트워크 | 다른 앱에 리워드 광고를 공급 | 매체 품질과 정산 구조가 중요 |

### 핵심 성과지표

| 지표 | 뜻 | 해석 포인트 |
|---|---|---|
| MAU / DAU | 월간 / 일간 활성 사용자 수 | 설치 수보다 실제 사용자를 봅니다. |
| DAU/MAU | 월간 사용자 중 매일 찾는 비중 | 높을수록 습관성이 강합니다. |
| 리텐션 | 가입 후 계속 쓰는 비율 | D1, D7, D30 기준을 자주 봅니다. |
| ARPDAU | 하루 활성 1명당 평균 매출 | 보상비 차감 전후를 구분해야 합니다. |
| CAC | 신규 사용자 1명 확보 비용 | LTV보다 낮아야 성장 여지가 있습니다. |
| LTV | 사용자 1명의 전체 이용 가치 | LTV > CAC 구조가 필요합니다. |
| 공헌이익 | 매출에서 보상·변동비를 뺀 잔액 | 리워드 앱 지속 가능성의 핵심입니다. |

### 리워드 앱의 단위경제

```text
광고·조사·제휴 매출
- 사용자 리워드
- 외부 매체 수익배분
- 현금화·상품권 수수료
- 부정사용 손실
- 변동 운영비
= 행동당 공헌이익
```

**챕터 요약:** 설치자 수가 많다는 말보다, 보상을 주고도 남는가가 본질입니다.

## 3. 주요 서비스 비교와 기업 사례

**핵심 메시지:** 사례마다 던지는 교훈이 다릅니다. 응답 품질, 반복 행동, 회수 구조, Web3 보상 설계, 보상비 통제를 따로 봐야 합니다.

| 서비스 | 유형·핵심 행동 | 보상·사용처 | 주요 수익모델 | 공개/추정 유저 지표 | 공개/추정 매출 지표 | 구조적 강점 | 주요 위험 |
|---|---|---|---|---|---|---|---|
| 마이비 | 취향 투표·퀴즈·걷기·게임·쇼핑 | 현금·네이버페이·상품권·금 | 광고·설문·오퍼월·쇼핑 제휴·애드체인 | 가입자 100만+·MAU 미공개 | 앱 단독 미공개 | 취향 데이터·W3ID·외부 매체 확장 | 보상비·응답 품질·개인정보 신뢰·수익성 |
| 오베이 | 타기팅 설문 응답 | 현금·네이버페이·기프티콘 | 기업 리서치·데이터 | 최근 4주 활성 패널 약 20만 | 오픈서베이 2024년 약 158억, 회사 전체 | 검증된 패널·품질 관리 | 불성실 응답·표본 편향 |
| 캐시워크 | 걷기·건강 챌린지 | 캐시·상품 교환 | 광고·커머스·B2B | MAU 450만 | 넛지헬스케어 2025 연결 1,182억, 앱 단독 아님 | 높은 빈도·대규모 트래픽·자체 광고기술 | 걸음 조작·광고 피로·보상비 |
| 토스 혜택 | 걷기·앱 실행·결제·제휴 | 토스포인트 | 금융·결제·광고·제휴 전환 | 만보기 1,150만명·주 2회+ 사용 | 혜택 단독 미공개 | 기존 금융 사용자·결제 데이터 | 대규모 보상비·구조 복잡성 |
| 리워디 | 리워드 지갑·코인/금 모으기 | 토큰·금·포인트성 보상 | 광고·미션·제휴·가상자산 연동 | Google Play 10만+ 다운로드·활성 미공개 | 앱 단독 미공개 | 지갑·보상·금 투자 경험 결합 | 가상자산 신뢰·보상 재원·규제/변동성 |
| 코퀴즈 | 퀴즈 참여·정답 보상 | 코넛코인·상금/이벤트 | 광고·토큰 생태계·NFT/커뮤니티 | 누적 다운로드 약 3만·누적 플레이 495만 | 앱 단독 미공개·상금 3억은 보상 원가 | 퀴즈형 참여와 토큰 보상 결합 | 토큰 유동성·보상비·지속 참여 |
| 돈이돼지 | 잠금화면·미션·걷기·쇼핑 | 현금성 포인트·기프티콘 | 광고·제휴 커머스·오퍼월 | DAU 11만+·Google Play 50만+ 다운로드 | 앱 흑자·비트바이트 연매출 100억+ 전망, 두 서비스 포함 | 일상 접점과 광고 재방문 구조 | 보상비·광고 피로·회사 전체 지표와 앱 지표 혼재 |

### 사례별 교훈

- **오베이:** 보상보다 응답 품질이 핵심입니다. 응답 수보다 광고주와 리서치 고객이 다시 구매할 만한 데이터 품질이 중요합니다.
- **캐시워크:** 매일 반복할 하나의 행동을 확보하면 광고와 커머스 확장 여지가 커집니다.
- **토스 혜택:** 리워드에서 손해가 나도 금융·결제·투자로 비용을 회수할 수 있는 슈퍼앱 구조가 강점입니다.
- **리워디:** Web3·자산형 리워드는 보상액보다 보상 자산을 어디서 쓰고 회수할 수 있는지가 신뢰를 좌우합니다.
- **코퀴즈:** 상금 규모는 매출이 아니라 보상 원가에 가깝고, 토큰형 리워드는 유동성과 반복 참여 설계가 함께 필요합니다.
- **돈이돼지:** 작은 보상도 매일 접점이 되면 매출 엔진이 될 수 있지만, 광고 피로와 보상비 통제가 중요합니다.

**챕터 요약:** 같은 리워드 앱이라도 강점, 위험, 수익 회수 방식은 제각각입니다. 마이비를 볼 때도 이 잣대를 그대로 적용해야 합니다.

## 4. 마이비의 사업 구조

**핵심 메시지:** 마이비는 퀴즈 앱에서 **취향 데이터 × 보상형 광고 플랫폼**으로 전환하는 단계입니다. 매출 성장은 확인됐지만 수익성·구조적 경쟁력은 아직 입증 전입니다.

초기에는 취향 퀴즈와 설문으로 B포인트를 주는 서비스였고, 현재는 만보기, 게임, 쇼핑, 출석, 현금 인출, 네이버페이/상품권, B포인트 적금, 금 기능까지 더한 종합 앱테크에 가깝습니다.

```text
사용자 취향 응답 수집
-> 사용자별 관심 정보 구성
-> 광고·조사 타기팅
-> 광고주로부터 수익 확보
-> 사용자에게 리워드 지급
```

### 공개 지표와 확인 필요 지표

| 항목 | 현재 판단 |
|---|---|
| 출시 시점 | 외부 보도는 2024-04, 운영사 연혁은 2024-05로 표기되어 확인 필요 |
| 가입자/다운로드 | 2025-04 가입자 100만 돌파 발표, Google Play 100만+ 다운로드 |
| 활성도 | MAU, DAU, D30 리텐션 미공개 |
| 현금화 조건 | 포인트 전환율, 최소 인출액, 출금비 확인 필요 |
| 금융형 기능 | 포인트 적금 재원, 금 수수료 확인 필요 |
| 앱 단독 수익성 | 공개 자료만으로 판단 불가 |

### W3ID와 SBT

마이비는 취향 정보를 **SBT**로 기록하고 이를 조합해 **W3ID**를 구성한다고 설명합니다.

사용자에게 실제 가치는 아래 형태로 나타나야 합니다.

1. 더 관련성 높은 설문과 광고
2. 같은 시간에 더 높은 보상
3. 내 데이터 사용 방식에 대한 확인과 통제

광고주에게는 직접 제공된 취향 정보가 잠재 장점입니다. 다만 응답이 부정확하거나 오래되면 광고 효과가 떨어질 수 있습니다.

### 수익모델과 재방문 장치

| 구분 | 내용 |
|---|---|
| 수익모델 | 보상형 광고, 설문·브랜드 퀴즈, 쇼핑 제휴, 오퍼월, 외부 앱용 광고 네트워크 |
| 애드체인 | 외부 앱에 퀴즈형 광고를 넣고 수익을 매체사와 배분하는 모델 |
| 재방문 장치 | 첫 달 보상, 짧은 퀴즈, 출석, 만보기, 출석패스, 방어권, 배지, 게임, 쇼핑, 포인트 적금, 금 기능 |
| 취약점 | 상당수 기능은 경쟁사가 복제하기 쉽고, 차별성은 취향 데이터의 실제 광고 성과에서 증명돼야 함 |

## 5. 마이비의 매출, 투자, 해석 주의점

원본 문서의 수치는 사람인이 NICE평가정보 기반으로 제공한 원셀프월드 전체 재무자료를 바탕으로 합니다. 감사보고서 원문 직접 확인값은 아니며, 마이비 앱 단독 실적도 아닙니다.

| 항목 | 2024 | 2025 |
|---|---|---|
| 매출 | 4억963만원 | 58억5,226만원 |
| 매출 증가율 | - | 계산상 약 +1,328.7% |
| 영업손실 | 18억9,414만원 | 35억1,707만원 |
| 당기순손실 | - | 33억4,759만원 |
| 영업이익률 | 계산상 약 -462.4% | 계산상 약 -60.1% |

### 해석할 때 주의할 점

| 포인트 | 해석 |
|---|---|
| 매출 성장 | 운영사 전체 매출 성장은 확인되지만 앱 단독 수익성 확보로 단정할 수 없음 |
| 손실 규모 | 영업손실은 확대됐지만 손실률은 개선된 것으로 계산됨 |
| 사업별 비중 | 광고, 애드체인, 쇼핑 제휴 등 비중 미공개 |
| 투자 | 크릿벤처스 2025-03 프리A 15억, 2025-09 후속 20억 등 합계 최소 35억 보도 |
| 운영사 발표 성과 | 가입자 100만, SBT 3,000만 발행, ARPDAU 경쟁사 4배+ 등은 비교 대상·산식·표본 확인 필요 |

**중요:** 마이비의 장기 사업성은 “매출이 늘었다”보다 **보상비를 지급하고도 캠페인별 공헌이익이 남는가**로 판단해야 합니다.

## 6. 마이비의 주요 위험요인

| 위험 | 내용 | 관찰해야 할 신호 |
|---|---|---|
| 보상비 증가 | 사용자 증가가 지급 포인트 증가로 이어짐 | 매출 증가보다 보상비·현금화 비용이 더 빠르게 늘어나는가 |
| 응답 품질 저하 | 보상 목적의 무작위 응답은 데이터 가치를 떨어뜨림 | 광고주 재구매율, 설문 품질 관리 기준 |
| 부정사용 | 다계정, 추천 조작, 자동화, 걸음 조작, 허위 전환 | 보류·회수·소명 정책의 명확성 |
| 개인정보 신뢰 | 데이터 소유와 대가를 강조할수록 동의와 설명 책임이 커짐 | 개인정보 처리방침, 제3자 제공, 추적 고지 |
| 광고 수요 변동 | 광고주 단가 하락 시 보상 축소나 자체 비용 부담 가능 | 광고 경기, 직접 광고 비중, 외부 매체 확장 |
| 정체성 약화 | 기능 확장으로 “왜 마이비인가”가 흐려질 수 있음 | 취향 데이터 강점이 화면에서 얼마나 유지되는가 |

## 7. 향후 전망 — 장기 성패를 가를 세 가지

마이비의 장기 성패는 아래 세 질문에 달려 있습니다.

1. **취향 타기팅 광고가 일반 광고보다 실제로 더 높은 성과를 내는가**
2. **보상을 주고도 캠페인별 공헌이익이 남는가**
3. **광고주와 외부 매체가 한 번이 아니라 반복해서 마이비를 쓰는가**

애드체인이 확대되면 자체 앱 사용자 수에 덜 의존하는 광고 네트워크로 성장할 수 있습니다. 반대로 효율이 일반 오퍼월과 비슷하거나 보상 축소 시 사용자가 빠르게 이탈하면 가격 경쟁에 빠질 위험이 있습니다.

**챕터 요약:** 마이비의 진짜 시험대는 가입자 수가 아니라 취향 데이터가 광고 성과로 증명되고, 보상 후에도 이익이 남고, 광고주가 반복 구매하는가입니다.

## 8. 리워드 앱 평가 체크리스트

### 사용자 가치

- [ ] 보상 가치가 사용자가 이해하기 쉬운 단위로 표시된다
- [ ] 현금화, 상품권 교환, 간편결제 전환 경로가 명확하다
- [ ] 최소 교환액, 수수료, 유효기간이 명확하다
- [ ] 사용자가 반복해서 돌아올 이유가 있다

### 사업성

- [ ] 다운로드가 아니라 DAU, MAU, 리텐션을 본다
- [ ] 매출에서 보상비와 변동비를 뺀 공헌이익을 본다
- [ ] 광고주가 반복 구매하는 구조가 있다
- [ ] 슈퍼앱처럼 리워드 외부에서 비용을 회수하는 경로가 있는지 확인한다

### 데이터 품질

- [ ] 보상 목적의 허위 응답을 줄이는 장치가 있다
- [ ] 응답 품질을 광고주에게 증명할 수 있다
- [ ] 취향 데이터가 오래되거나 부정확해질 때 갱신하는 방식이 있다
- [ ] 개인정보 동의와 제3자 제공 고지가 명확하다

### 리스크

- [ ] 다계정, 자동화, 추천 조작, 걸음 조작 대응 정책이 있다
- [ ] 보상 축소 시 사용자 이탈 가능성을 본다
- [ ] 회사 전체 실적과 앱 단독 실적을 구분한다
- [ ] 운영사 발표치와 독립 검증된 사실을 구분한다
