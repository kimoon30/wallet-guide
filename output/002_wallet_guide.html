핵심 포인트 4가지 — 먼저 이것만

01

월렛은 보관함이 아니라 ‘리모컨’

코인은 앱 안이 아니라 블록체인 장부의 내 주소 항목에 있고, 월렛은 그것을 움직이는 키와 서명을 관리합니다.

02

주소는 공개, 키는 비공개

지갑 주소는 공개해도 되지만, 개인키·시드 문구는 누구에게도 알려주면 안 됩니다. 노출되는 순간 통제권을 잃습니다.

03

기능이 아니라 ‘유형’으로 구분

① 키를 누가 관리하나 ② 어떤 체인을 지원하나 ③ 어떻게 복구하나 ④(dApp이라면) 어떤 권한·지역 제한이 있나.

04

리워드 앱은 ‘공헌이익’이 본질

성패는 다운로드 수가 아니라 보상 지급 후 남는 공헌이익입니다. 매출이 늘어도 보상비가 더 빨리 늘면 적자입니다.

🟦비전공자는 먼저 세 가지만

  • 고를 때는 기능 수보다 키 관리 주체, 지원 체인, 복구 방식을 먼저 봅니다.
01

시작 전 가드레일 초급

Guardrails Before You Start

핵심 메시지 · 개념을 이해하는 것과 실제 자산을 옮기는 것은 다릅니다. 실습은 ‘잃어도 되는 소액’으로 시작합니다.

이 문서를 읽고 할 수 있게 되는 것과 아직 조심할 것을 먼저 구분합니다.

✓ 읽고 나면 할 수 있는 것

  • ✓월렛이 코인을 담는 앱이 아니라 키를 관리하는 도구임을 설명
  • ✓지갑 주소·개인키·시드 문구·가스비·커스터디 차이 구분
  • ✓대표 서비스 6종의 유형 차이 읽기
  • ✓거래 승인 팝업에서 주소·수량·수수료·권한 확인

✕ 아직 조심해야 할 것

  • 큰 금액을 바로 전송하거나, 처음 보는 사이트에 지갑 연결
  • 시드 문구를 클라우드·메신저·고객센터·웹 입력창에 입력
  • 지원 체인·복구 방식·수수료 코인·지역 제한 확인 없이 이동
  • 내용이 불분명한 서명 요청을 습관적으로 승인
⚠️처음 실습할 때의 원칙

처음에는 학습 비용으로 감당할 수 있는 소액만 사용합니다. 본 거래 전 같은 네트워크에서 소액 테스트 전송을 먼저 해봅니다. Ethereum.org도 지갑을 “블록체인 계정에 접근하는 도구”로 설명하며, 사용자가 거래 승인과 복구 정보를 스스로 관리해야 한다고 안내합니다.[6]

02

디지털 월렛이란 무엇인가 초급

What a Digital Wallet Actually Is

핵심 메시지 · 월렛은 자산을 저장하는 곳이 아니라, 자산을 움직일 권한(개인키)을 관리하는 도구입니다.

한 문장 정의

디지털 월렛은 블록체인에 기록된 내 디지털 자산을 확인·전송·사용할 수 있도록 ‘암호학적 열쇠’를 관리하는 도구입니다. 코인·NFT가 앱 안에 파일처럼 저장되는 것이 아니라, 자산과 거래 내역은 블록체인에 기록되고 월렛은 그 자산을 움직일 개인키를 관리합니다.

쉬운 비유 — 월렛은 ‘열쇠 꾸러미’에 가깝다

비유실제 개념
공동 장부블록체인 — 모든 소유·이동 기록이 적힌 장부
우리 집 공개 주소지갑 주소 — 남에게 알려줘도 되는 식별값
문을 여는 비밀 열쇠개인키 — 거래에 서명하는 권한
열쇠 꾸러미·도어록 앱디지털 월렛 — 주소와 열쇠를 관리
장부상 보유 자산코인·토큰·NFT

주소를 아는 것은 ‘집 주소를 아는 것’과 비슷합니다. 주소만으로는 자산을 옮길 수 없지만, 개인키를 가진 사람은 거래에 서명할 수 있으므로 사실상 그 지갑을 통제합니다.

잘못된 송금기관을 통한 조치 가능성중앙기관이 없어 되돌리기 어려운 경우 많음

단, 모든 월렛에서 사용자가 키를 직접 관리하는 것은 아닙니다. 회사가 대신 관리하는 월렛도 있으며, 이 차이가 커스터디 / 논커스터디입니다(03장).

📌NIST의 월렛 정의 (출처 보강)

NIST의 NISTIR 8301(2021)은 ISO 22739:2020 정의를 인용해, 월렛을 “개인키와 공개키를 생성·관리·보관·사용하는 애플리케이션”으로 설명합니다.[1]

03

꼭 알아야 할 핵심 용어 초급–중급

Core Terminology

핵심 메시지 · 모두 외울 필요 없습니다. “어디에 기록되나(블록체인) → 어떤 자산인가(코인/토큰/NFT) → 누가 승인하나(개인키/시드 문구)” 순서로 이해합니다.

블록체인 · 네트워크(체인)

  • 블록체인 — 여러 컴퓨터(노드)가 함께 보관하는 분산 거래 장부. 거래를 묶은 ‘블록’이 이전 블록의 해시로 연결돼 변조가 매우 어렵습니다. 은행은 ‘은행을 믿어서’, 블록체인은 ‘규칙과 다수 합의로’ 신뢰가 성립합니다.
  • 네트워크(체인) — 블록체인은 하나가 아닙니다. 비트코인(가치 저장), 이더리움(스마트 계약), 솔라나(빠르고 저렴) 등 각자 규칙·수수료가 다른 독립 네트워크입니다. 각 체인의 자산은 기본적으로 다른 체인으로 바로 이동할 수 없습니다 → 월렛이 어떤 체인을 지원하는지 꼭 확인해야 하는 이유.

스마트 계약

조건이 충족되면 중개자 없이 자동 실행되는 블록체인 프로그램입니다(자판기 비유). 토큰·NFT 발행, DeFi 대출, 탈중앙화 거래소(DEX)가 모두 이 위에서 작동합니다. 월렛의 “서명 요청” 팝업이 바로 스마트 계약과 상호작용하는 순간입니다.

⚠️주의

스마트 계약은 코드이므로 버그·악의적 설계가 있을 수 있습니다. 내용을 확인하지 않고 서명하면 자산 인출 권한을 악성 계약에 넘겨줄 수 있습니다.

04

월렛은 실제로 어떻게 작동하나 중급

How It Works — Scenarios

핵심 메시지 · 모든 거래는 ‘주소 확인 → 수량/수수료 확인 → 개인키 서명 → 네트워크 기록’이라는 같은 골격을 따릅니다.

시나리오 A — MetaMask로 ETH 보내기

  1. 받는 주소 입력 — 0x…. 한 글자만 틀려도 자산이 사라질 수 있어 복사·붙여넣기 사용.
  2. 수량 입력
  3. 가스비 확인 — 수수료(ETH) 표시. 낼 ETH가 없으면 전송 불가.
  4. 확인(서명) — 이 순간 개인키로 거래에 디지털 서명.
  5. 네트워크 전파 → 노드 검증 → 블록 기록 — 상대 주소 잔액이 장부에서 증가.
내 지갑 주소 → 개인키로 서명 → 네트워크 전파 → 노드 검증 → 블록 기록 → 상대 잔액 증가

핵심 — MetaMask는 서명 도구, 네트워크는 장부, 개인키는 권한 증명. 코인은 지갑 ‘안’이 아니라 블록체인 장부의 내 주소 항목에 있습니다.

시나리오 B — NFT 마켓에서 NFT 구매하기

  1. 월렛 연결 — 주소만 사이트에 공개됩니다. 개인키는 절대 전달되지 않습니다.
  2. NFT 선택(예: 0.05 ETH)
  3. 거래 승인 팝업 — NFT 가격 + 가스비 합계 확인(예: 0.053 ETH 필요).
  4. 서명 및 전송 — 스마트 계약이 ① 내 ETH를 판매자에게 전송 ② NFT 소유권을 내 주소로 이전.
  5. 소유권 기록 — 장부의 소유자가 내 주소로 변경.
지갑 연결(주소 공개) → 구매 승인(개인키 서명) → 스마트 계약 실행 → NFT 소유권 이전

What Reward Apps Are

핵심 메시지 · 리워드 앱은 무료로 돈을 주는 곳이 아니라, 기업이 원하는 행동의 대가 일부를 사용자에게 지급하는 사업입니다.

사용자가 보상을 받는 행동은 광고 시청·설문/퀴즈·걷기·출석·앱 설치/가입·쇼핑·친구 초대·게임 등. 포인트는 현금·상품권·간편결제·할인 등으로 사용. 예를 들어 쇼핑 앱이 신규 회원 1명에 5,000원을 지불하면, 플랫폼은 그중 일부를 사용자에게 주고 나머지로 운영비·수수료를 충당합니다.

사용자 관점기업 관점
남는 시간·일상 행동을 소액 가치로 전환. 보상까지 시간이 짧고, 포인트 가치가 명확하고, 현금화가 쉽고, 반복 이유가 있어야 함돈을 나눠주려는 게 아니라 광고 노출/클릭, 설치/가입, 설문 응답, 쇼핑·금융 유도, 재방문, 행동 데이터 수집을 위해 운영
📌리워드 앱의 핵심 사업 질문

“기업이 사용자 행동으로 얻는 매출이, 사용자에게 지급하는 보상과 운영비보다 큰가?” 여기에 계속 ‘그렇다’고 답할 수 있어야 사업이 유지됩니다.

돈과 데이터의 흐름

[돈]   광고주 캠페인 의뢰 → 플랫폼이 미션 제공 → 사용자 수행
       → 광고주가 결과에 따라 비용 지급 → 일부를 보상·제휴 비용으로
       → 남은 금액으로 인건비·서버비·마케팅비·수익
[데이터] 사용자의 취향·구매·반응 → 플랫폼 분석 → 누구에게 어떤 광고를 보일지 결정

타기팅이 정확할수록 광고주는 같은 비용으로 더 많은 성과를 얻습니다. 다만 보상 때문에 사실과 다르게 답하면 데이터 가치가 떨어집니다.

챕터 요약

보상의 재원은 광고·조사·제휴 비용입니다. 그래서 ‘데이터 품질’과 ‘반복 참여’가 사업의 생명선입니다.

  • 돈이돼지 — 작은 보상도 매일 접점이 되면 매출 엔진이 됩니다. 공식 페이지는 DAU 11만+·누적 적립금 60억원+을 제시하고, DARO 사례는 앱 흑자와 비트바이트의 연매출 100억원+ 전망을 설명합니다. 교훈: 보상비를 통제하려면 광고 위치, 세션 빈도, 정산 구조가 함께 맞아야 합니다.[21]

핫 월렛 / 콜드 월렛인터넷 연결 소프트웨어 / 키를 오프라인 보관
핫 월렛은 인터넷에 연결된 앱·브라우저 확장 형태로, 쓰기 편한 대신 해킹에 노출됩니다(예: MetaMask). 콜드 월렛은 열쇠를 인터넷과 분리된 기기에 오프라인으로 보관해 훨씬 안전합니다(예: 하드웨어 지갑). 보통 자주 쓰는 소액은 핫에, 오래 보관할 거액은 콜드에 둡니다.
임베디드 월렛앱 내부에 내장·연동된 지갑(예: 마이비)
별도 지갑 앱을 따로 설치하지 않아도, 일반 앱 안에 지갑 기능이 내장된 형태입니다. 사용자는 복잡한 시드 문구나 가스비를 거의 의식하지 못한 채 평소 쓰던 앱처럼 이용할 수 있습니다(예: 마이비). 블록체인을 모르는 사람도 쉽게 진입하도록 장벽을 낮춘 방식입니다.
dApp / 스마트 월렛월렛 연결해 쓰는 탈중앙화 앱 / 스마트 계약 기반 월렛
dApp(탈중앙화 앱)은 회사 서버가 아니라 블록체인 위에서 돌아가는 앱으로, 지갑을 연결해 로그인·거래합니다. 스마트 월렛은 단순 열쇠가 아니라 스마트 계약으로 만든 지갑이라, 지출 한도 설정·복구 도우미 지정 같은 똑똑한 기능을 넣을 수 있습니다.
Web3 / EVM지갑으로 신원·자산을 통제하는 인터넷 / 이더리움 호환 실행 환경
Web3는 플랫폼이 내 계정·자산을 쥐고 있는 기존 인터넷과 달리, 지갑으로 내 신원과 자산을 스스로 통제하는 인터넷을 가리킵니다. EVM(이더리움 가상 머신)은 이더리움이 프로그램을 실행하는 표준 환경으로, 'EVM 호환' 체인들끼리는 같은 지갑·앱을 거의 그대로 쓸 수 있습니다.
거래 해시 / 블록 탐색기거래 조회 번호 / 온체인 기록 검색 도구
거래 해시는 보낸 거래마다 붙는 고유 영수증 번호입니다. 블록 탐색기는 그 번호를 넣으면 거래가 언제·얼마·어디로 처리됐는지 누구나 조회할 수 있는 공개 검색 사이트입니다(예: Etherscan). 송금이 실제로 됐는지 직접 확인할 때 씁니다.
Polygon / pUSD이더리움 호환 저수수료 체인 / Polymarket 담보 토큰(USDC 기반)
Polygon은 이더리움과 호환되면서 수수료가 훨씬 싸고 빠른 블록체인입니다. pUSD는 Polymarket에서 베팅 담보로 쓰는 토큰으로, 가치가 1달러에 고정된 USDC를 기반으로 합니다. 가격 변동 없이 '현금처럼' 쓰기 위한 안정(스테이블) 토큰입니다.
XPLA마이비 트랜잭션·지갑이 처리되는 메인넷
마이비의 거래와 지갑이 실제로 처리되는 메인넷(독립된 블록체인)입니다. 사용자가 마이비 안에서 보상을 받거나 자산을 옮길 때, 그 기록이 이 XPLA 네트워크에 남습니다.
SBT양도 불가하도록 지갑에 귀속된 토큰(취향 뱃지 등)
SBT(소울바운드 토큰)는 한번 받으면 다른 사람에게 팔거나 넘길 수 없도록 내 지갑에 묶이는 토큰입니다. 매매 대상이 아니라 '이 사람의 자격·취향·이력'을 증명하는 뱃지·증명서 용도로 쓰입니다(예: 수료증, 취향 뱃지).
MAU/DAU·리텐션월간/일간 활성 사용자 수·재사용 비율
MAU는 한 달 동안, DAU는 하루 동안 앱을 실제로 쓴 사람 수입니다. 리텐션(재방문율)은 새로 들어온 사용자가 며칠 뒤에도 다시 돌아오는 비율로, 앱이 얼마나 '계속 쓸 만한지'를 보여 주는 건강 지표입니다.
ARPDAU·CAC·LTV활성 1명당 매출·신규 1명 확보 비용·1명 생애 가치
ARPDAU는 활성 사용자 1명이 하루에 만들어 주는 평균 매출입니다. CAC는 신규 사용자 1명을 데려오는 데 든 비용(주로 광고비), LTV는 한 사용자가 떠날 때까지 평생 만들어 주는 총 가치입니다. LTV가 CAC보다 커야 사업이 남습니다.
공헌이익매출에서 보상·변동비를 뺀 실질 잔액
매출에서 사용자 보상·결제 수수료처럼 '팔 때마다 따라붙는 변동비'를 뺀 실질 잔액입니다. 여기서 다시 고정비(인건비·임대료 등)를 빼면 최종 이익이 됩니다. 리워드 앱은 보상이 곧 변동비라, 공헌이익 관리가 특히 중요합니다.
타기팅·오퍼월·CPA맞춤 광고·미션 목록 광고·행동 완료 시 과금
타기팅은 사용자 성향에 맞춰 광고를 골라 보여 주는 것입니다. 오퍼월은 "앱 설치하면 포인트" 같은 미션 목록 형태의 광고 지면입니다. CPA는 단순 클릭이 아니라 가입·구매 같은 행동이 실제로 완료됐을 때만 광고비를 내는 과금 방식입니다.
