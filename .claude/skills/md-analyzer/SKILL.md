# Markdown 분석 스킬

## 1. 구조 파싱

마크다운 원소별로 추출한다:

| 원소 | 추출 항목 |
|------|----------|
| 제목 | H1~H4 텍스트, 깊이, 순서 |
| 문단 | 연속 텍스트 블록, 단어 수 |
| 목록 | 순서/비순서, 최대 중첩 깊이, 항목 수 |
| 표 | 열 수, 행 수, 헤더 여부 |
| 코드 | 언어 태그, 줄 수 |
| 이미지 | src URL, alt 텍스트 존재 여부 |
| 인용 | 텍스트, 출처 표시 여부 |

## 2. 목적 분류

다음 단서로 목적을 추론한다:
- H1 제목의 어조 (개요·가이드·보고서·발표 등 키워드)
- 전체 길이와 섹션 수
- 표·데이터 비중 vs 설명 비중
- 코드 블록 비중
- 글머리 기호 vs 서술 문장 비율

**분류:**
- `리포트`: 분석·비교·데이터 중심, 긴 설명
- `발표`: 핵심 메시지 위주, 섹션당 내용 적음
- `가이드`: 단계별 지시, 코드 블록 많음
- `참고자료`: 정의·목록·링크 중심

## 3. 정보 밀도 측정

- **낮음**: 평균 문단 100단어 미만, 섹션당 핵심 개념 2개 이하
- **보통**: 100~200단어, 2~4개 개념
- **높음**: 200단어 초과, 5개 이상 개념 또는 다수 전문 용어

## 4. 갭 식별 패턴

다음 패턴을 탐지한다:

| 유형 | 탐지 방법 |
|------|----------|
| 근거없음 | "~이다", "~해야 한다" 주장에 근거 문장 없음 |
| 출처불명 | 숫자·퍼센트·통계가 있지만 출처 표시 없음 |
| 정의없음 | 약어·전문 용어 첫 등장 시 설명 없음 |
| 비교기준없음 | "A보다 낫다", "더 빠르다" 주장에 기준 없음 |
| alt없음 | `<img>` 또는 `![…](…)`에 alt 텍스트 비어있음 |

## 5. 레이아웃 추천

**PPT형** (아래 4가지 모두 충족 시):
- 문서 목적이 "발표"
- 전체 단어 수 < 1500
- H2 섹션 4개 이상
- 섹션당 평균 단어 수 < 200

**리포트형**: 위 조건 하나라도 불충족 시 기본값.

판단이 애매하면 리포트형을 선택하고 `layout_reason`에 "경계 케이스로 기본값 선택" 명시.

## 출력

`_workspace/01_analysis.json`에 분석 결과를 저장한다. 형식은 `md-analyzer.md` 에이전트 정의의 출력 스키마를 따른다.

분석 완료 후 `SendMessage`로 **content-enricher**에게 전달:
```
"분석 완료. 파일: _workspace/01_analysis.json. 갭 {N}개 식별됨."
```
